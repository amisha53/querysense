Why I built this

In CS 440 I kept writing queries that worked but couldn't always articulate why — especially with complex joins and subqueries. QuerySense is the tool I wished existed during that class. It breaks down what each part of a query actually does and catches things like missing indexes that slow everything down in production.

Learning SQLUnderstand what GROUP BY vs HAVING actually does, and when to use each.
Code reviewQuickly audit a query someone else wrote before running it on production data.
DebuggingPaste a slow query and immediately see which parts are likely causing performance issues.
Interview prepVerify your understanding of how a query executes before explaining it out loud.
